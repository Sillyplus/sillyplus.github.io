Mac配置Vim折腾小纪录

Sep 16, 2014 in 技术

环境：Mac OS X 10.10， VIM - Vi IMproved 7.3

虽然说最近其实一直都是在使用Atom，可是不知道为什么每过一段时间，总会有各种原因激起我使用vim的热情。。。前几天看到了rrr好像在用着一个看起来还挺不错的vim，跟他要配置，后来他给了我一个网址：https://github.com/ma6174/vim，在这里感谢一下作者的无私奉献。

其实这个配置的作者基本上已经帮我们完成了所有的任务，我们要做的就是打一条命令，然后让它自己安装就行了，事实上我也是这么做的。但是在经过一小段漫长的等待时间之后，我看到的却是这个

Error detected while processing /Users/silly/.vim/plugin/client.vim:

line  314:

Traceback (most recent call last):

  File "<string>", line 2, in <module>

  File "/Library/Python/2.7/site-packages/Twisted-14.0.0-py2.7-macosx-10.10-intel.egg/twisted/__init__.py", line 53, in <module>

    _checkRequirements()

  File "/Library/Python/2.7/site-packages/Twisted-14.0.0-py2.7-macosx-10.10-intel.egg/twisted/__init__.py", line 37, in _checkRequirements

    raise ImportError(required + ": no module named zope.interface.")

ImportError: Twisted requires zope.interface 3.6.0 or later: no module named zope.interface.

Press ENTER or type command to continue

突然觉得看不懂，好烦不想看，就直接按了回车，进入了vim，感觉虽然报错了，可是用起来并没有什么问题啊，
至少我写写那几行小代码是这样子的，可是仔细一想我又觉得虽然没什么影响的样子，可是每次打开vim的时候都
有这个报错也是挺令人觉得烦的，你们说对吧，然后我也懒得看是什么原因了，直接暴力重装了几次，可是都还是
不行，
只好滚回来自己看报错的内容咯，看了一下好像没看懂是什么原因，但是发现最后有这么一句

ImportError: Twisted requires zope.interface 3.6.0 or later: no module named zope.interface.

想可能是python的某个什么东西的某个东西(zope.intergface)有什么问题吧，问了一下度娘。然后要在mac下安装zope.interface，可以先到这里下载 https://pypi.python.org/pypi/zope.interface。下载gz压缩包那个版本，解压后运行一下里面的setup.py，在terminal打开到解压后的文件夹，运行

sudo python setup.py install

然后很快就能安装好了。然后问题就因此解决了，然后我就能快乐的跟vim玩耍一阵子了～
